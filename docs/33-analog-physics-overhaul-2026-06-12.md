## 2. 재구현된 물리

### 진공관 단 (TubeStage)
- **플레이트 노드 암시적 해석**: `Cp·dVp/dt + warm·Ip(Vp,Vgk) + (Vp−Vb)/Rp = 0` 을 워밍업 포함 Newton(웜스타트 2회)으로 샘플별 해석. rp∥Rp 압축·포화측 클리핑·플레이트 스트레이 캐패시턴스의 비대칭 슬루가 물리로 발생 (기존 ad-hoc 슬루 블록은 기본 OFF).
- **CF 암시적 캐소드 해석**: `warm·Ik(Vb−Vk, VG−Vk) = Vk/Rk + Ck·dVk/dt` — 발진 완치, 팔로워 게인 ≈ µ/(µ+1).
- **정지점 자기일관 해석**: CC/CF/펜토드/LTP 전부 로드라인 포함(펜토드는 단조성 이분법). `outputMakeup_`이 레거시 소신호 게인을 복원해 레벨 보정은 유지, 곡률만 물리화.
- **그리드 전도**: 관 자체의 Dempwolf Ig 연속 법칙(접촉전위 −0.5 V 시프트) + 스토퍼 소스 로딩 Newton — 양(+) 피크 실시간 플래트닝과 블로킹 충전이 같은 전자로 일원화. 정지 그리드-리크 평형 프라이밍.
- **Miller**: 해석된 동작점의 |Av| = gm·(Rp∥rp) 사용, 포화 시 톱이 열리는 문서 방향. 소스 임피던스는 체인이 전달하는 **이전 단의 순시 출력 임피던스**.
- 1/f 플리커 노이즈(Kellet), 펜토드 파티션 노이즈 √(1+2·Ig2/Ip), 히터 험 2차 고조파, DC 트래커 0.5 Hz 고정(fs 독립).

### 펜토드 (KorenPentode)
- **전하 보존 분할**: Ik = Ip + Ig2 항상 성립 — 플레이트가 못 거둔 전류는 스크린이 수집(니의 스크린 새그 압축 루프가 굶지 않음). 2차 방출은 플레이트→스크린 이동(보존).
- **서프레서 = 수집 게이트**: 공간전류가 아닌 plateShare를 변조 — 하드 서프레서 바이어스에서 Ig2 상승(실측 6AS6 방향). CV PentodeHigh: 스크린 87 V→58 V 드룹 확인.
- EF86 muG2 8→38, 6AS6 muG2 7→28 (g1→g2 µ), G/G2 데이터시트 재피팅. 스크린 공급이 B+ 레일을 추종(새그/리플/래더가 펜토드에도 도달).

### 트랜스포머 (TransformerStage) — 션트 손실 구조
- **권선비 유니티 전달 − 비선형 자화 전류의 소스 강하**: `v_out = v_in − k_loss·H_mag − k_eddy·dΦ/dt`, `Φ += v_out`, `H_mag = JA⁻¹(B(Φ))` (상태 복사 시컨트 역해석, 웜스타트).
  - 선형 영역에서 루프 극점이 정확히 Lm 코너(fc_low) — 삽입 게인 ±0.7 dB 유니티 (χ 의존성 소멸).
  - 자속 ∝ 1/f → **저역 우선 포화** (50 Hz 압축 ≫ 2 kHz, 신규 테스트로 가드).
  - 와전류 = dΦ/dt 션트(f² 손실), 재질별 계수(Si-steel 0.05 … 아모퍼스 0.006).
  - **DC 비대칭 포화**: SE는 300B 대기 전류의 표준 자화(H_dc, 갭 코어), PP는 잔여 불균형 전류를 셋업에서 주입 — docs/02 §6.
- 5개 모드 OPT 드라이브를 실측 노드 레벨 기준 코어 여자 0.5–0.7a로 재보정 (CV는 의도적 과여자 경로 유지).

### 전원부 (PowerSupplySag + 체인 래더)
- **정류기+리저버 실회로**: |cos| 전파정류 파고 근방 도통 펄스 충전(R_rect = Z/3) + 부하 방전. 새그 진폭/비대칭 회복/부하 비례 톱니 리플(I/(2fC))/관 vs SS 정류기가 전부 창발. 빈티지 리저버 22–47 µF.
- **단별 B+ 디커플링 래더**: 각 단의 노드가 **프리셋의 Vp_nominal에 정확히 안착하도록 드로퍼 R 자동 보정** — 문서화된 동작 전압이 실제 정상상태가 됨. 입력단일수록 리플 필터링 깊고(실기처럼 프리앰프는 조용), 출력단 대전류가 래더를 타고 상류 노드를 ~0.5 s 시정수로 끌어내리는 "레일을 통한 단간 대화"가 물리로 발생.
- Marshall 레일 325→450 V(JCM800 EL34 명세), HiFi 360 V — PP/300B가 문서 동작점에서 실제로 동작.

### 푸시풀 (PushPullStage)
- OPT **DC(DCR 80 Ω)/AC(반사 3.4 kΩ) 부하선 분리**: 대기점은 레일 근방, 신호는 AC 부하선을 타고 레일 위로도 스윙(인덕티브 플라이백).
- 테일 강하 기준 바이어스 관행(Vg_bias = 정지 Vgk) — EL34 idle 55 mA/측(데이터시트 앵커), driveScale 7 V = 풀스케일이 정확히 class-A/AB 경계.
- 플레이트 소산 기반 열 드리프트(파워단 "주저앉음") 추가.
- **바이어스 조정 모델링**: 시드별 µ 산포가 idle을 지수적으로 흔드는 것을, 실제 정비 관행대로 idle 목표 이분법으로 Vg 재트림 — 시드 차이가 레벨(3.3 dB)이 아닌 캐릭터(곡률/미스매치/AB 전이)로 발현. Marshall 시드 널 −1.1 → −22.3 dB.

### Monte Carlo (ComponentVariation)
- 입력/출력 트랜스포머·PP·확장엔진 **독립 시드 추출** (기존: 첫 프리앰프 단 복제).
- 전해콘 로그정규 비대칭(−20/+80%), 저항 시대별 σ(빈티지 카본 5%/워른 8%/와일드 12%), 페르베언스 σ 12%(Dempwolf 실측 산포), 양수 물리량 로그정규(부호 반전 방지), 펜토드 법칙/워밍업 τ/페어 미스매치/리저버 C 변동 추가.

### 신호 경로 품질
- **오버샘플러**: 탭 수 64·F+1 (팩터 비례) + 진짜 폴리페이즈 분해 — 통과대역 20 kHz < 0.3 dB, 기저 나이퀴스트 ≥ 90 dB, 모든 팩터에서 라운드트립 지연 63샘플 균일(OS 전환 시 PDC 불변).
- **TP 리미터**: 검출 FIR 군지연 자동 유도·보상, 룩어헤드 슬라이딩 미니멈(모노토닉 덱) + 0.1 ms 어택 — ISP 온셋 천장 위반 제거.
- 체인 인터랙션/피드백/LF 엔벌로프 시정수 fs 정규화 (48 kHz 기준 재매핑).
- 테이프 엔진: JA 산화철 히스테리시스(자속 구동, 주파수 의존 포화), 옵토: T4 트랩 레벨 메모리(τ₃ ≈ 25 s, 릴리즈 ~3× 신장), 뉴럴 잔차 DC 차단(5 Hz).

## 3. 보정 관행

물리 교체로 음량/배음 보정이 무효화되는 것을 막기 위해 세 가지 보정 상수를 도입했다 — 모두 셋업 시 해석적으로 계산되며 곡률(왜곡 형태)에는 관여하지 않는다:

1. `TubeStage::outputMakeup_` — 레거시 소신호 게인 / 물리 게인 비.
2. `PushPullStage::loadLineMakeup_` — 동일 원리 (AC 부하선 슬로프 기준).
3. 트랜스포머는 구조적으로 유니티(권선비)라 보정 불요 — 대신 5개 프리셋의 `drive`를 코어 여자 기준으로 재기록.

## 4b. 고강도 코드리뷰 후속 수정 (2026-06-13)

7관점 멀티에이전트 리뷰로 확정/유력 19건을 잡아 수정:

- **와전류 손실 부호 오류** (critical): `vOut = u − k·vOut[n−1]` 형태가 나이퀴스트에서 역위상이 되어 의도한 손실이 1/(1−k) **HF 부스트**로 뒤집힘. 병렬 손실저항의 순시 분배기 `vOut = (u − kLossH·H)/(1+k_e)`로 교체.
- **TP 리미터 바이패스 크로스페이드 누출** (high): `bypass` 플래그가 켜지는 즉시 리미터가 우회되는데 bypass 크로스페이드(5 ms)는 램프 중이라, 그 창에서 wet 경로(레벨매치×출력×A/B 게인 스택 최대 ~64배)가 **리미터 없이** 출력에 섞여 +30 dB 초과 스파이크 발생(OPT를 삽입-유니티로 바꾼 뒤 wet 피크가 커져 노출됨). wet이 블록에 기여하면 리미터를 유지하도록 수정 — 완전 dry 블록만 리미터 해제(트루 바이패스 비트일치 유지).
- **TP 룩어헤드 하한** (high): 0.1 ms 어택 τ는 fs에 비례하나 기본 룩어헤드는 64샘플 고정 → 192 kHz에서 브릭월 보증 깨짐. 어택 τ의 12배로 룩어헤드 하한 설정.
- **펜토드 Miller/Zout 동결** (high): 펜토드 분기가 `lastGmLoaded_/lastRpInvLoaded_`를 갱신하지 않아 CV 모드의 동적 Miller·소스 임피던스가 셋업 시점 삼극관 프록시 값에 고정. 잔차 슬로프와 Langmuir gm∝√I로 매 샘플 갱신.
- **CF 그리드 전도** (high): 캐소드 팔로워는 캐소드가 그리드를 따라가 Vgk가 바이어스 근처에 묶이는데, 전도 법칙이 그리드 드라이브로 평가돼 발생 불가능한 블로킹 차지 누적. Vk 오프셋으로 Vgk 기준 평가.
- **비바이패스 CC 이중 디제너레이션**: 언바이패스 Rk 경로가 i·Rk를 빼면서 동시에 캐소드 바운스 deltaVk도 빼 게인 절반. rest-기준 단일 경로로 통합.
- **트랜스포머 오버라이드가 보정 삭제** (high): UI/CLI에서 다른 철심 선택 시 drive/H_scale/outputGain/H_dc까지 스톡값으로 덮어써 HiFi 출력 +13 dB 폭주·코어 8배 과여자. 철심(코어/대역/공진)만 교체하고 체인 보정은 유지하도록 수정.
- 기타: CLI 확장엔진 MC 누락 연결, 리롤 크로스페이드 시 레일 래더 스냅샷, DryDelay 링 128→256(63샘플 지연 4배 여유), `decouplingR` 사용 연결, JA 역해석 커밋 중복 RK4 제거, 그리드 전류 해석적 도함수, 스크린 RC alpha 셋업 호이스트, LTP 중복 평가 제거, 펜토드 캐소드 자기바이어스에 스크린 전류 포함, pink 필터 reset, CI fit-pipeline 타임아웃 45→90분.

### 회귀 수정 (수정 검증 중 발견, 2026-06-13/14)

리뷰 수정을 회귀 검증하다 OPT를 "포화 리미터"에서 "삽입-유니티"로 바꾼 결과 노출된 두 가지 잠재 결함을 추가로 잡았다:

- **JilesAtherton 서브스텝 무상한 루프 → 오디오 스레드 행** (critical): `step()`의 적응 서브스텝(|ΔH|>a/4 시 분할)에 상한이 없어, V72 체인 출력이 테이프 자속 적분기를 강하게 구동해 H가 거대해지자 샘플당 수천 회 RK4 서브스텝이 돌며 expansion 테스트가 25분 행. 오디오 스레드였다면 드롭아웃. 서브스텝 상한 64 추가(|Mirr|≤Ms 클램프가 결과를 물리적으로 보장하므로 병리적 과도에서 약간의 정확도만 양보하고 실시간 보장). 테스트 25분→0.06초.
- **PushPull 시동 DC 스텝 → flux OPT 증폭** (high): 비대칭 페어의 정지 출력(diff)이 0이 아닌데 `outputDC_`가 0에서 시작해, 2초짜리 0.5 Hz AC-커플러가 그 스텝을 제거하는 동안 flux OPT(DC 차단 ~10ms 코너)가 통과시켜 시동 시 +37 dB 스파이크. `outputDC_`를 정지 정규화 출력으로 프라이밍(setup/reset) → 72→18로 감소.
- **출력 안전 소프트클립** (전이 한정): 브릭월 리미터가 꺼지는 순간(바이패스/TP off·soft/null-test) 리미터 룩어헤드 딜레이 링이 limit 이전 핫 콘텐츠를 무제한 배출하는 구조적 구멍을, 전이 구간(리미터 모드 변화·재구축 페이드·바이패스 크로스페이드)에만 게이팅되는 C1-연속 소프트클립(±1 이하 비트항등, +6 dBFS 점근)으로 차단. 정상 정상상태·트루 바이패스·limiter-off 레퍼런스 경로는 게이팅으로 완전 비활성 → 비트일치 유지.

이 수정들로 plugin stress 스위트(rapid-toggle/TP-mode/AB-toggle/bypass-transparency)가 -j4 병렬·반복 실행에서 안정적으로 통과한다.

### 3차 검토 (6각 적대 검토, 2026-06-14)

추가로 실제 결함 5건을 잡아 수정:

- **그리드-차지 평형 프라이밍이 잔여 zeroing에 덮어써짐** (high): setup() 말미의 `gridChargeFastV_=gridChargeSlowV_=0`이 직전 평형 프라이밍을 무효화. 체인은 로드/리롤/단 편집마다 reset()이 아닌 setup()으로 재구축되므로, 매번 22 ms 블로킹-회복 시동 과도가 재주입됐다. 잔여 zeroing 제거.
- **JilesAtherton 서브스텝 상한 64가 큰 H 램프에서 RK4 오버슛** (회귀): 1차 수정의 상한 64가 오프라인 히스테리시스 램프(ΔH≈1e5)에서 dH_sub를 거대하게 만들어 M이 −Ms로 뒤집힘. **적용 필드를 ±64·a로 포화 클램프**(그 너머는 M=±Ms로 물리적 무손실)하고 백스톱 캡 600으로 교체 — 정당한 적분은 정확하고 런어웨이만 묶인다.
- **PushPull NaN 복구가 정지 오프셋 대신 0으로 리셋** (med) + **dissAvg NaN 미복구** (low): 둘 다 setup()/reset()과 동일하게 프라이밍하도록 수정.
- **소프트클립 hold이 블록 단위라 작은 호스트 버퍼에서 링 드레인 미보장** (low): 샘플 단위(최대 룩어헤드 바닥값 + 2블록)로 교체.
- 검증된 무결성: 확장엔진 MC가 플러그인·CLI 양 경로 동일, 리롤 스냅샷 완전성, JA 클램프가 refit 하모닉 프로파일 불변(refit 재통과로 확인).

회귀 부작용 2건(JA-reset, cold-start-differs)도 근본 원인(콜드스타트 차이는 hot V72 saturation에 가려져 비포화 레벨에서 측정)을 파악해 함께 정리. 최종: **ctest 176/176** (refit 포함), python 검증 4종, stress 스위트 -j4 ×2 안정 통과.
