## 1. P0 — 정합성 결함 (코드가 자신의 물리와 모순)

### 1.1 PushPull 내부 LTP 위상반전기에 로드라인이 없다

`PushPullStage::processLtpPhaseSplitter()` / `setupLtpPhaseSplitter()` ([PushPullStage.h:518-613](../src/plugin/../dsp/PushPullStage.h))는 양측 삼극관을 **고정 플레이트 전압** `ltpVpNominal`에서 평가한 뒤, 사후에 `Vp = VpNominal − Ip·Rp`로 스윙을 만든다. 이것은 docs/33 §1 결함 #1("플레이트 로드라인 미해석 — 정적 무부하 곡선, 게인 gm·Rp")과 **정확히 같은 오류**가 PP 경로 안에 남은 것이다. `TubeStage`의 LTP 토폴로지는 수정됐지만 이 복제본은 누락됐다.

- **증상:** 스플리터의 곡률/헤드룸이 비물리 (rp∥Rp 압축 없음, 포화측 클리핑 없음). `ltpToPowerGridGain = 0.22`가 과대 게인을 레벨만 눌러놓은 상태라, Console 모드에서 Drive를 올릴 때 위상반전기가 실물보다 이르고 딱딱하게 접힌다. 출력 클램프 `±80 V`(608-612행)는 이 비물리 게인의 방어막이다.
- **보강:** `TubeStage`의 LTP 경로와 동일한 per-side 플레이트 Newton(웜스타트 2회) + 테일 결합을 이식. `loadLineMakeup_`과 같은 방식으로 `ltpToPowerGridGain`을 셋업타임 재보정(레거시 소신호 게인비)해 레벨 불변·곡률만 물리화. 수렴 후 ±80 V 클램프는 NaN 가드로만 남긴다.
- **비용:** side당 삼극관 평가 2회 추가 — Marshall 프리셋 CPU +2% 내외.

### 1.2 그리드 노드의 소스 임피던스가 경로마다 다르고, 그리드 네트워크가 모드 무차별이다

같은 물리 노드(그리드)를 두 코드 경로가 서로 다른 소스 임피던스로 본다:

- **Miller 필터** ([TubeStage.h:1798-1801](../src/dsp/TubeStage.h)): `zSrc = 이전 단 rp∥Rp (또는 config.sourceImpedance)` — **그리드 스토퍼 미포함**. V72 S1 기준 fc ≈ 1.3 MHz로 사실상 비활성. 스토퍼 68 kΩ을 포함하면 fc ≈ 12 kHz — 스토퍼의 존재 이유(입력 용량과의 RC) 그 자체가 빠져 있다.
- **그리드 전도 Newton** ([TubeStage.h:919-951](../src/dsp/TubeStage.h)): `rStop`만 사용 — **업스트림 Zout 미포함**. 반대 방향의 누락.

여기에 더해 **그리드 네트워크 3종(`gridStopperR` 68 k, `gridCouplingC` 22 n, `gridLeakR` 1 M)은 어떤 프리셋도 오버라이드하지 않는다.** 결과:

| 모드 | 인터스테이지 (실제 회로 선언) | 블로킹 네트워크 (실제 사용값) | 불일치 |
|---|---|---|---|
| V72 | 22 n / 1 M | 22 n / 1 M / 68 k | 스토퍼만 의문 |
| Console | 22 n / **470 k** | 22 n / **1 M** / 68 k | 회복 τ 2× |
| CV | **100 n** / 1 M | **22 n** / 1 M / 68 k | 충전량 4.5× 과소 |
| HiFi | **220 n / 470 k** | **22 n / 1 M** / 68 k | 회복 τ 5× 상이 |

즉 모든 모드의 "블로킹/스퀴시" 시상수와 양피크 플래트닝 강도가 **자기 회로가 아니라 Marshall풍 전역 기본값**으로 동작 중이다.

- **보강:**
  1. `R_src_total = dynamicSourceZ + gridStopperR`을 Miller fc와 전도 Newton **양쪽에 공통 적용** (전도 쪽은 `Ig·(rStop + zSrc)` 강하).
  2. 프리셋별 그리드 네트워크 실화: 인터스테이지 선언값과 단일 출처로 통일(1.4와 함께), 스토퍼는 회로 관행대로 — 브로드캐스트/HiFi 1–10 k, 기타앰프 계열 33–68 k, CV 드라이브단 낮게.
  3. 음색이 움직이므로(특히 하드 드라이브 플래트닝 깊이) refit 게이트 1회 필수.
- **비용:** 0 (연산 동일, 상수만 정합).

### 1.3 오극관 전하 보존에서 Ig1이 빠져 있다

`KorenPentode::evaluate()` ([KorenPentode.h:154-180](../src/dsp/KorenPentode.h))는 `Ik = Ip + Ig2`를 보존하지만 **양그리드 전도 시 `Ig1`이 공간전류에서 차감되지 않는다** (실물: `Ik = Ip + Ig2 + Ig1`). CV 하드 드라이브(양피크에서 g1 전도)에서 그리드가 훔친 전자가 장부에 없다. 같은 이유로 `TubeStage`의 캐소드 바운스 Ik 합산([TubeStage.h:1310-1317](../src/dsp/TubeStage.h))에도 Ig1 몫이 없다.

- **보강:** `evaluate()`에서 분할 전 `Ispace' = max(0, Ispace − Ig1)`로 차감(전하 보존 완결), 캐소드 전류 합산에 Ig1 포함. 소신호 무영향, 하드 드라이브에서만 미세 차이 → refit 게이트에 무리 없음.

### 1.4 같은 커플링 캡이 두 곳에서 서로 다른 값·서로 다른 MC 추첨으로 존재한다

인터스테이지 HPF는 `interstageCc × coupling_scale`로 ([TubeAmpChain.h:615-627](../src/dsp/TubeAmpChain.h)), 블로킹 충전 네트워크는 `gridCouplingC × gridCouplingC_scale`로 ([TubeAmpChain.h:1310-1311](../src/dsp/TubeAmpChain.h)) — **동일 물리 부품이 결정값도(1.2 표) 확률 추첨도 따로 논다.** 한 유닛 안에서 "22 nF 캡의 HPF 코너"와 "그 캡이 블로킹 전하를 담는 용량"이 다른 부품이 되는 셈.

- **보강:** `TubeStageConfig.gridCouplingC`를 단일 출처로 삼고 인터스테이지 HPF가 그 값(다음 단의 것)을 읽도록 배선 + MC 스케일 1회 추첨 공유. 1.2의 프리셋 실화와 한 패치로 처리.

### 1.5 한 인스턴스 안에 mains 기준이 3개 있고 하나는 60 Hz 하드코드다

- PSU 리플 위상 (`PowerSupplySag`, 시드 산포) — 2×line
- 히터 험 위상 (`TubeStage`, 단별 산포) — 1×line + 2f 성분
- 출력단 리얼리즘 험 ([PluginProcessor.cpp:2124-2135](../src/plugin/PluginProcessor.cpp)) — **60 Hz 하드코드**, L/R 주파수 1.003배 디튠

실물에서 이들은 전부 **같은 전원 트랜스**에서 나온다: 히터 픽업 위상은 배선 임의(단별 산포는 물리적으로 정당)지만, 정류 리플의 크레스트와 히터 사이클, 출력 누설 험은 강체 위상 관계다. 또한 `heaterFrequency`(단 설정)와 `psu.ripple_freq`(120 고정), 출력 험(60 고정)이 각각 따로라 50 Hz 지역 세팅이 일관되게 불가능하다.

- **보강:** 인스턴스당 **mains 위상 누산기 1개**(시드로 초기 위상 산포)를 체인이 소유. 히터 = mains(+단별 고정 오프셋), 정류 드라이브 `|cos(mains)|`, 출력 누설 험 = mains(+오프셋). 전역 50/60 Hz 스위치 1개로 파생 주파수 전부 유도. 비용 0(발진기 수는 오히려 감소).

### 1.6 마이너 정리

- `LinearTransformerColor` ([TubeAmpChain.h:141-180](../src/dsp/TubeAmpChain.h)) — 사용처 없는 데드 코드. 삭제.
- CV의 서프레서 구동이 `|vgSignal|` 엔벌로프 ([TubeStage.h:1023-1029](../src/dsp/TubeStage.h)) — 실물 CV의 g3는 DC BIAS 노브. 현 구조는 "신호 적응형" 보이싱으로 유효하되, Bias 파라미터가 `suppressorBiasVolts`를 직접 움직이는 경로(진짜 노브 물리)를 병행 노출 검토.
- MC에서 튜브 G·mu가 독립 추첨 — 실물은 지오메트리로 약한 양의 상관. `corr ≈ 0.4` 코퓰러 한 줄이면 되고 시드 널 특성이 더 실물다워짐 (낮은 우선순위).

## 2. P1 — 문서(docs/04)가 약속했으나 코드에 없는 물리

33 정비로 "단 내부"는 물리가 됐다. 남은 큰 덩어리는 전부 **블록 경계**에 있다. 아래 5개는 docs/04의 해당 절이 이미 메커니즘을 문서화해 놓았다.

### 2.1 전역 부귀환(NFB)이 실루프가 아니다 — docs/04 §5

현재 `feedbackAmount`는 출력의 저역/고역 스무딩 상태를 입력에서 빼는 **엔벌로프 보이싱** ([TubeAmpChain.h:953-966, 1154-1179](../src/dsp/TubeAmpChain.h))이다. 실물 NFB의 핵심 물리 — *출력단이 클리핑하는 순간 루프 게인이 소멸하면서 무릎이 급경화되고, 그 직전까지는 THD와 출력 임피던스를 루프 게인만큼 억제* — 는 구조적으로 나올 수 없다.

- **보강:** OPT 출력(정규화 신호)을 β 네트워크(RC 분압: 저역 β_lo + presence 고역 감쇠)로 **샘플 피드백**해 첫 파워단 입력(Console: 스플리터 입력, HiFi 옵션: 300B 드라이버 캐소드)에 감산. OS 레이트에서 1샘플 지연 루프이므로 β·A_loop < 수십 dB 범위에서 안정 (48 k×4 기준 지연 5.2 µs — 실물 배선 지연과 같은 차수). 셋업 시 체인 소신호 게인 A를 해석해 목표 NFB dB(예: Console 10 dB, HiFi 6 dB, V72 0 dB — 실물 V72는 국부귀환 중심)로 β 자동 산출, 닫힌 루프 게인은 `outputMakeup` 관행대로 레벨 보정.
- **창발 검증 가드:** (a) 소신호 THD가 β dB만큼 감소, (b) 클립 온셋에서 knee 기울기 급증(무귀환 대비), (c) 저역 댐핑(출력 임피던스) 감소.
- 기존 `feedbackAmount` 보이싱 레이어는 실루프 도입 후 deprecated 경로로 격하.

### 2.2 OPT 자화 전류가 플레이트로 되먹지 않는다 — docs/04 §6.1

`TransformerStage`는 자화 전류 `H_mag`를 계산해 **자기 출력에서만** 강하시킨다 ([TransformerStage.h:334-336](../src/dsp/TransformerStage.h)). 하지만 그 자화 전류는 실물에서 **플레이트가 공급**한다 — 저역·대신호일수록 튜브가 반사 부하 외에 자화 전류까지 감당하며 로드라인이 기울어지고(§6.1), 저역 헤드룸이 먼저 소진된다. 현재는 "OPT가 저역을 먼저 포화시키는 것"(33에서 확보)은 있지만 "그 포화가 튜브를 더 힘들게 만드는 것"이 없다.

- **보강 (1단계, 저비용):** OPT가 매 샘플 노출하는 `i_mag_norm = H_mag/(H_scale·drive)`를 체인이 읽어, 다음 샘플 PP/SE 플레이트 노드 KCL에 부하 전류로 가산 (1샘플 지연 결합 — PSU 래더와 같은 패턴). 환산 계수는 셋업 시 정규화 체계에서 해석적으로 유도.
- **보강 (2단계, 완전판):** 자화 브랜치를 플레이트 노드의 companion(상태: Φ, 기울기: JA 증분 투자율 1샘플 지연)으로 넣어 2.5와 통합.
- **효과:** 대신호 베이스에서 파워단 전류·새그·열 드리프트가 함께 반응 — "저역을 밀 때 앰프 전체가 버티는" 실물 거동이 전 계통(레일→바이어스→OPT)으로 연결된다.

### 2.3 Class-AB 부하 불연속(Raa/2 ↔ Raa/4)이 없다 — docs/04 §7

`solveSide()`는 측별 **고정** `rAc = Rp_primary`를 쓴다 ([PushPullStage.h:366-399](../src/dsp/PushPullStage.h)). 실물 PP에서 한쪽이 컷오프하면 남은 관이 보는 반사 임피던스가 Raa/2 → Raa/4로 **절반으로 꺾인다** — AB 전이의 로드라인 킹크는 크런치 스펙트럼의 구조적 성분이고, 두 반쪽 프라이머리가 자기적으로 결합돼 있어 컷오프 측 플레이트가 상대 스윙을 그대로 뒤집어쓰는(2×B+ 플라이백) 것도 같은 물리다.

- **보강 (근사, 권장):** 각 측 `rAc_eff = (Raa/4)·(1 + w_partner)`, `w_partner = 상대측 전도 비중`(Ip_partner/Ip_rest 기반 0..1 스무딩) — class-A에서 Raa/2, 깊은 AB에서 Raa/4로 연속 이동. 비용: 곱셈 2개.
- **보강 (구조, v2):** 두 플레이트 노드를 OPT 차동 제약으로 묶은 2×2 Newton — 플라이백 파형까지 정확해짐. 근사판 검증 후 결정.
- **가드:** Drive 스윕에서 AB 진입 시 H3/H5 성장 기울기의 실측형 꺾임 재현 여부.

### 2.4 파워단에 그리드 전도/블로킹이 없다 — docs/04 §4.2의 파워단 적용

프리앰프 `TubeStage`에는 Dempwolf Ig 연속 법칙 + 2-τ 블로킹 메모리가 있으나 `PushPullStage`의 파워 그리드에는 **아무것도 없다**. 오버드라이브된 실물 PP에서 가장 큰 "회복 물리"는 파워관 그리드 전도가 스플리터 커플링 캡을 충전해 바이어스를 끌어내리는 것(크로스오버 시프트, 듀티 워크, blocking fart)이다. Console 모드가 Drive ≥ 2에서 "British crunch"를 광고하는 만큼 이 영역의 핵심 메커니즘 부재다.

- **보강:** `TubeStage`의 grid-charge 네트워크(연속 Ig + fast/slow 브랜치)를 side당 이식. 소스 임피던스 = 위상반전기 rp∥Rp (1.1 수정 후 자연 제공). 전도 영역 밖 스킵 최적화 동일 적용 — 평시 비용 ≈ 0, 크런치 시 side당 Newton 2회.

### 2.5 반사 부하가 순저항이다 — docs/04 §6.2 (로드라인의 주파수 의존)

`Rp`/`Rp_primary`는 스칼라 저항. 실물 부하는 리액티브다: HiFi 300B의 스피커(모션 공진 수십 Hz에서 |Z| 수배 상승, 보이스코일 L로 고역 상승), V72/RNDI의 600 Ω 라인+케이블 용량. 로드라인이 주파수에 따라 타원으로 열리며 **클리핑 비대칭이 주파수의 함수**가 되는 것 — SE 파워단 소리의 큰 축이다.

- **보강:** 플레이트 Newton의 부하항을 companion 어드미턴스로 일반화. 이미 `capG`(Cplate) companion이 있는 구조라, 부하를 `G0 + G_L(상태1) + G_C(상태2)` 3-엘리먼트(R + 직렬RL + 병렬C, 트래페조이드 이산화)로 확장하면 된다. 프리셋: HiFi = 8 Ω 스피커 등가(공진 45 Hz Q 1.5, L_vc 0.5 mH 반사), Console = 라인 부하, V72/RNDI/CV = 저항 유지(기본 무변화).
- **비용:** 상태 2개 + Newton 잔차에 항 2개 — 해당 단만 +3% 내외.

## 3. P2 — 디바이스 · 전원 물리 보강

### 3.1 그리드 전류의 플레이트 전압 의존 (전류 분할)

Dempwolf Ig는 `Ig(Vgk)`뿐이다. 실물은 **Vp가 낮아질수록(플레이트가 스윙 바닥) 그리드가 공간전류의 더 큰 몫을 훔친다** (Rutt 1984의 division 영역) — 딥 오버드라이브에서 블로킹 충전이 현행보다 세다. 보강: `Ig_eff = Ig(Vgk)·[1 + k_div·σ((Vg−Va)/V_div)]` 소프트 스위치 1개 (해석 도함수 유지). CV/Console 하드 드라이브의 "빨려들어가는" 회복감 개선.

### 3.2 JA 동적 손실 (Bertotti excess loss)

현행 와전류는 병렬 저항의 평탄 분배기(1차 물리로 정확, docs/33 §4b)지만, 실측 코어는 **루프 폭 자체가 주파수와 함께 커진다**(보자력 ∝ √f — 초과 손실). 보강: `k_dyn = k·(1 + κ_exc·√(|ΔH|/a))`를 `dMirrDH` 분모에 반영 (재질별 κ_exc: Si-steel 큼, 아모퍼스 ≈ 0). 고역 대신호에서 이력 손실이 실물처럼 증가 — "iron이 HF에서 뻣뻣해지는" 질감. 비용: sqrt 1회/서브스텝.

### 3.3 정류관의 3/2승 법칙 + B+ 기동 램프

`R_rect` 상수 선형 충전 ([PowerSupplySag.h:233-237](../src/dsp/PowerSupplySag.h))을 Child-Langmuir로: `I_rect = P·max(0, vDrive−Vc)^{3/2}` (P는 기존 Z_internal 스펙에 맞춰 셋업 보정). 대전류에서 도통 강하가 초선형으로 커져 **새그의 깊이-비선형성**(살짝만 밀면 안 주저앉고, 세게 밀면 확 주저앉는)이 실물화된다. 함께: 정류관 히터 τ(~11 s)로 `vPeak_` 기동 램프 → Warmup 버튼이 "gm 페이드"가 아니라 **레일이 차오르며 동작점이 자리잡는** 실물 시퀀스가 된다 (SS 프리셋은 즉시 기동 + 오버슛).

### 3.4 초크/π-필터 2차 공진

RC 래더는 새그가 단조 회복만 한다. 초크 입력 실물은 L-C 공진(수–수십 Hz, Q < 1)으로 **새그 회복이 살짝 오버슛**("bounce")한다 — GZ34+초크 계열의 펌핑 필이 여기서 나온다. 보강: 첫 래더 노드에 인덕터 상태 1개(선택형, GZ34/5U4 프리셋만). 비용: 곱셈 3개.

### 3.5 Miller 피드스루 영점

현행 Miller는 입력 극점만 모델. 실물 Cgp는 클리핑 순간 **플레이트 에지를 그리드로 관통**시킨다(피드포워드 영점) — 하드클립 에지의 "유리질 스핏". 보강: `vg += Zsrc_total·Cgp·dVp/dt` (1샘플 차분, 1.2의 Zsrc 통일 후 자연 정의). 소신호 무영향.

### 3.6 커플링 캡 누설 — 빈티지 바이어스 드리프트 MC 축

종이/왁스 캡의 누설 저항(수십–수백 MΩ)은 이전 단 플레이트 DC를 다음 그리드로 분압시켜 **유닛마다 바이어스가 조금씩 양(+)으로 밀린** 상태를 만든다 — "같은 채널스트립인데 채널마다 성깔이 다른" 고전적 원인. 보강: `Vg_bias_eff += V_plate_prev_rest·Rg/(R_leak+Rg)`, `R_leak`은 Vintage/Worn/Wild 분포에서 로그정규(Modern은 ∞). 셋업타임 계산이라 비용 0. 시드 널이 레벨이 아닌 곡률·비대칭으로 벌어지는 좋은 축.

### 3.7 카본 저항 초과 잡음 (분포 연동)

카본 컴포지션 저항은 DC 전류에 비례하는 1/f 초과 잡음을 낸다 (금속피막의 10–30 dB 위). 현행 1/f 플리커는 캐소드 계면만 모델. 보강: Vintage/Worn/Wild에서 `Rp` 양단 DC 강하에 비례하는 미세 1/f 전압 잡음 추가 — "빈티지 유닛일수록 바닥이 살아 있는" 분포 차등. 기존 feel-verify 마이크로모션 게이트(≤0.9) 준수 확인 필수.

### 3.8 다음 단 Rg가 이전 단 AC 부하에 미반영

CC 플레이트의 AC 부하는 실물에서 `Rp ∥ (Rg_next + …)` — 1 MΩ Rg면 −9% 게인과 곡률 변화, 470 kΩ(Console/HiFi)이면 더 크다. CC 솔버는 이미 DC/AC 분리 구조(`plateLoadIsTransformer` 경로)가 있으므로: 일반 CC도 `vOpen = rest 앵커 + AC 슬로프 = Rp∥Rg_next`로 확장. refit 게이트 1회.

### 3.9 트랜스포머 왜곡의 구동 임피던스 의존

`kLossH_`는 셋업 고정 ([TransformerStage.h:212-213](../src/dsp/TransformerStage.h)) — 실물 트랜스 THD는 **구동 소스 임피던스에 비례**한다(자화 전류가 소스 저항에서 만드는 강하가 곧 왜곡 성분). 튜브 rp가 신호에 따라 출렁이는 것이 트랜스 왜곡을 실시간 변조하는 상호작용이 현재 없다. 보강: 체인이 이전 단 `lastOutputImpedance()`를 트랜스에 주입, `kLossH_eff = kLossH·(Z_dyn/Z_setup)` (1샘플 지연, 클램프 0.5–2.0). Lm 코너도 함께 미세 이동 — 물리적으로 옳은 방향.

## 4. P3 — 아키텍처 블라인드스팟

### 4.1 볼트-네이티브 체인 (단간 전압 연속성)

현행 단간 인터페이스는 `plate V → ÷normalizer → ×makeup → ×outputGainLinear → 다음 단 ×inputVoltageSwing → grid V`다. 33 정비로 **단 내부 곡률**은 물리가 됐지만, **단 사이의 드라이브 관계**(1단이 2단을 그리드 전도로 얼마나 미는가)는 여전히 프리셋 상수의 산물이다. 또한 단마다 0.5 Hz DC 트래커 + 7 Hz 인터스테이지 HPF의 **이중 AC 커플링**이 있는데 실물 네트워크는 하나다.

- **목표 상태:** 단간 신호를 **볼트 그대로** 전달. 커플링 캡 1개가 (a) HPF 코너, (b) 블로킹 전하, (c) DC 차단을 전부 담당(1.4의 단일 출처화가 선행 조건). 정규화·레벨 보정은 체인 입출력에서 각 1회 (`chainMakeup` — 정책의 셋업타임 보정 관행 그대로).
- **마이그레이션:** ① `VoltageBus` 경로를 플래그로 병행 구축 → ② 프리셋별 소신호 등가(±0.1 dB) + 하모닉 프로파일 등가(±1 dB) 자동 검증 → ③ refit 재생성 → ④ 구 상수 제거. 이 리팩터가 끝나면 1.2/1.4/3.8이 구조적으로 재발 불가능해진다.

### 4.2 확장엔진/뉴럴의 OS 배치 (docs/33 §5 잔여 과제의 구체화)

`ExpansionRack`(특히 Tape JA — 강한 이력 비선형)과 `NeuralResidualLayer`가 다운샘플 이후에 동작 ([PluginProcessor.cpp:2038-2047](../src/plugin/PluginProcessor.cpp)). 테이프 경로의 고조파가 기저 나이퀴스트에서 접힌다. 권장: 전체 재배선 대신 **Tape 경로만 자체 2× 경량 OS 랩**(짧은 half-band, 상태 2벌) — 옵토/FET는 게인 스무딩이라 base-rate 무해, 뉴럴은 7 kHz LP 방어가 이미 있어 유지.

### 4.3 자동화 시 상태 이월 (docs/33 §5 잔여 과제의 구체화)

Bias/Drive/스테이지 편집이 `rebuildChain()`을 타며 느린 상태(워밍업·열·그리드 차지·JA 이력·래더)를 리셋한다. 권장: `TubeStage::rebaseTo(const TubeStage& old)` — 새 rest 기준으로 상대 상태(워밍업 진행률, ΔVk, 블로킹 전하−rest, ipAvgLong−rest, JA state 통째)를 이월. 크로스페이드 경로는 이미 있으므로 "새 그래프가 이월 상태로 시작"만 추가하면 된다.

### 4.4 기존 v2 항목 유지 확인

울트라리니어 탭(43%), 인터스테이지 트랜스포머 토폴로지(docs/14)는 v2 유지. 단, 2.3(구조판)과 UL 탭은 같은 "결합 OPT 노드" 작업이므로 v2에서 묶어 설계할 것.

- **3.7 카본 저항 초과 잡음** — `excessNoise_offset`을 스테이지 flickerNoiseRatio에 가산(Modern 0, Vintage 0.10 / Worn 0.22 / Wild 0.40 중심, 총합 ≤0.85 클램프 — feel-verify 마이크로모션 예산 내).
- **G-µ 상관(1.6 잔여)** — 공통 성분 구성 `g = √ρ·z_c + √(1−ρ)·z_i`(ρ=0.4)로 perveance와 µ가 같은 캐소드 로터리를 타도록 — **marginal σ는 정확히 보존**(모집단 가드: σ_µ≈0.08, σ_G≈0.12, ρ∈[0.15,0.65] 확인).

**부수 발견·수정:** NFB 프로브(`measureForwardGain`, 테스트 전용)의 settle 1024샘플(21ms)이 0.5 Hz DC 트래커·PSU 충전·JA 정착에 태부족 — 일부 MC 추첨에서 시동 과도가 RMS를 최대 ~6× 과대측정(해석 추정치와의 대조 게이트가 이를 노출; 블록별 격리로 스테이지·PP 모델은 전부 실측 일치 확인). settle을 2.5 s(레이트 파생)로 수정 → 측정/추정 비 1.21×.

**검증:** ctest **193/193**(신규 가드 2종: 시대 병리 등급/캡, 상관+marginal 보존) · refit 재생성 → 게이트 재확인.

### W6 — 아키텍처 4.2 + 4.3 (2026-07-12, 완료)

M/S 플레이크 12/12 통과로 검증.

### W7 — v2 항목 (2026-07-13, 완료)

### W8 — 4.1 볼트-네이티브 체인 (2026-07-13, 완료)

§4.1의 4단계 마이그레이션을 완료했다:

- **① 병행 구축** — `TubeStageConfig.voltageNativeInput/Output` + `TubeAmpChainConfig.voltageNativeInterface`: RC 경계의 단간 신호가 **플레이트 볼트**로 흐르고(`volts → 커플링 HPF → ×pad → 다음 그리드 볼트`), pad는 셋업에서 레거시 보정곱(`outputGainLinear·makeup·swing/normalizer`)으로 합성되는 **명시적 단간 어테뉴에이터**. 인터스테이지-트랜스포머 경계는 정규화 유지(철심 드라이브 캘리브레이션이 정규화 도메인).
- **핵심 물리 변화** — 내부 스테이지의 출력이 **정적 rest 앵커** 기준 볼트가 되어 0.5 Hz DC 트래커가 출력을 더 이상 소거하지 않음 → 새그·열로 인한 동작점 이동(0.5–7 Hz)이 커플링 캡을 타고 **다음 그리드의 바이어스를 실제로 펌핑**(실기의 "숨쉬는 동작점"). 이중 AC 커플링(트래커+RC)이 물리 1개(RC)로 정리됨. 트래커는 미터용으로 유지.
- **② 등가 게이트** — 정상상태 V72: H1 ±0.1 dB·H2/H3 ±1 dB 이내(패드 합성이 선형 핸드오프를 비트 동일하게 재현), 대신 대신호 버스트 후 회복 궤적은 두 모드가 유의하게 분기(펌핑 경로 생존 가드 >1e-3) — 의도된 유일한 차이가 물리라는 증명.
- **③ 기본 ON 전환 + refit** — 전 프리셋 volt-native로 전환, 전체 스위트 199/199, refit 재생성.
- **④ 레거시 경로** — `voltageNativeInterface=false` 폴백 플래그로 1사이클 유지 후 제거 예정. 구 상수들은 삭제가 아니라 **물리적 패드 값**으로 의미가 재정의됨(실기 콘솔의 단간 패드와 동형) — D1이 지적한 "정규화 핸드오프의 비물리성"은 해소, 1.2/1.4/3.8 계열 정합성 이슈의 구조적 재발 방지 달성.
